# Universal Save System (USS) for Unity

一个基于 **Zenject (DI)**、**异步 IO** 与**反射自动化**实现的高性能、高可靠性 Unity 存档框架。

## 🌟 核心特性

* **原子级文件操作 (Atomic Save)**：采用 `Temporary File + File.Replace` 机制，确保存档过程即便遭遇程序崩溃或意外断电，也不会损坏原有存档数据。
* **自动化字段映射 (Auto-Reflection)**：通过 `[DataList]` 特性标记，系统在启动时自动扫描并初始化 `SaveBlob` 数据结构，省去手动注册不同数据类型的繁琐过程。
* **读写分离架构 (RO/WO Separation)**：
    * **RO (Read-Only)**：提供 `IReadOnlyList` 视图，防止业务层意外篡改基础数据。
    * **WO (Write-Only)**：集中管理数据变更，确保数据流向清晰。
* **异步磁盘 I/O**：序列化与文件写入操作脱离主线程（`Task.Run`），在大规模数据存储时依然能保持丝滑的帧率。
* **高性能反射缓存**：在静态构造函数中预处理字段信息，将反射损耗降至最低。

## 🏗️ 架构组件

### 1. DataStore (持久化层)
负责底层磁盘交互。
* **安全保障**：使用 `.tmp` 中转文件，写入成功后进行原子替换。
* **目录自愈**：自动检测并创建缺失的存档路径。

### 2. DataRepo (数据仓库)
系统的“内存数据库”，负责管理运行时的所有数据列表。
* **类型安全**：通过泛型约束确保所有存储对象均遵循 `IDatable` 协议。
* **运行时镜像**：在初始化时创建数据的运行时拷贝，实现存档与逻辑层的有效隔离。

### 3. DataManager (接口层)
作为系统的唯一对口外观（Facade），集成了 Zenject 的 `IInitializable`，实现场景加载时的自动存档检索与分发。
